                Thread actions
                

                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                  
                

                

                  Actions dispatch to the live Codex thread. New thread is the only one that takes effect without an active task — the rest require an already-running Codex session (start any task first). Goal actions use Codex's experimental persisted goal API. Errors surface as a toast + Activity log entry.
                

              

              

                Sandbox
                

                  
                  
                

                

                  workspace-write blocks listener sockets / pipes and outbound network. Prefer danger-full-access only when a task genuinely needs live app automation (UNO, D-Bus) or arbitrary network.
                

              

              

                Approval policy
                

                  
                  
                

                

                  untrusted: only run known-safe commands without approval. on-request: model asks when it wants to. never: execute without asking (only with an external sandbox).
                

              

              

                Model
                

                  
                  
                

                
Sent as model in thread/start. Empty = let Codex pick its configured default.

                

                  
                  
                

                
Maps to -c model_reasoning_effort=.... Only meaningful for reasoning-capable models; ignored otherwise.

              

              

                Tools

                Sandbox permissions
                

                  
                  
                  off
                

                

                  Only applies when sandbox is workspace-write. Maps to
                  sandbox_workspace_write.network_access. Middle ground between the default and danger-full-access when a task needs outbound HTTP but no listener binds.

                Session actions
                

                  
                

                

                  Gemini latches every knob below at process spawn. New session forces a restart with the current settings even if nothing has changed. Other slash commands (/compact, /chat, /memory) aren't exposed over ACP — start a new session instead when context gets crowded.
                

              

              

                Approval mode
                

                  
                  
                

                

                  default: prompt for every tool call. auto_edit: auto-approve edits, prompt for exec. yolo: auto-approve everything. plan: read-only (no writes or exec).
                

              

              

                Sandbox
                

                  
                  
                  off
                

                

                  Maps to gemini --sandbox. Constrains shell execution and file writes. Leave off when you need the same environment as your terminal.
                

              

              

                Model
                

                  
                  
                

                
Sent as gemini --model. Empty = let Gemini CLI pick its configured default.

              

              

                Extensions
                

                  
                  
                

                

                  Passed as gemini --extensions=a,b,c. Empty = use all installed extensions (Gemini's default). List installed ones with gemini extensions list.
                

              

              

                MCP server allowlist
                

                  
                  
                

                

                  Maps to gemini --allowed-mcp-server-names. If you set an allowlist, include intendant or the Intendant MCP tools (take_screenshot, etc.) won't be reachable. Empty = all servers allowed.
                

              

              

                Workspace
                

                  
                  
                

                

                  One absolute path per line. Maps to gemini --include-directories. Blank lines and duplicates are dropped. Press Ctrl/⌘+Enter or click away to save.
                

              

              

                Debug
                

                  
                  
                  off
                

                

                  Maps to gemini --debug. Opens Gemini's Chromium DevTools console. Mostly useful for tracking down Gemini CLI itself.
